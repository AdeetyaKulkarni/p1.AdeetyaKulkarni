Answers for B:

Questions regarding microwave interface
  • List your most common uses and other more rare uses of a microwave. Does the interface make doing those common things easier?

    • Most common uses : Heating food, Timed cook, Defrost
    Least common uses : Easy clean (helps clean the utensils by heating sticky food), One touch cooking (For Pizza, Popcorn etc), Weighted sensor cooking (senses the weight of the food item and cooks it)

    The interface does not make doing the common things easier -

    1. The [Cancel | Clear] and [Start | Add 30 sec] buttons are clubbed together which can confuse the user who is not familier with the UI.
    2. The express cook functionality is vague & non intuitive to the user
    3. The UI has the Timer and Time Cook buttons next to each other which may confuse the user.
    4. The defrost functionality has too many options and it is unclear how each option works.

  • Think about how the user interacts with the microwave. What is the common sequence of actions?

    • The sequence of user interaction with the microwave takes place as follows -

    1. User opens the microwave door, places the food in the center and closes the door
    2. Select the option to execute and press the required button/buttons to start
    3. The microwave starts â€“ This is indicated by the light inside the microwave
    4. Once the task is done microwave provides a feedback (Beeping sound)
    5. User opens the microwave door takes food out and closes the door again.

  • How does the microwave support the user figure out how to make it work?

    • The microwave supports the user as follows -

    1. Labels - The microwave has labels like "Start" or "Add 30 sec" to specify the functionality
    2. Feedback - The microwave provides feedback by lights and sounds so tell the user the end of an action
    3. Symbols - Microwave has some symbols like "oz" or a cup symbol to explain what is being measured
    4. Digital display - The microwave diplays the selected user action in the digital diplay to make sure the user is doing what they want.

  • How does the microwave provide feedback to the user?

    1. Light inside the microwave - ON (if door open or something is heating) | OFF (if machine is idle)
    2. Digital display on interface - Displays digits | Labels for certain options
    3. Sound - Buzzer sound to indicate a procedure is done | Sound to indicate button is pressed

  • What are some common mistakes you can make with this current design?

    1. You might accidentally start express cook when clicked on a button without pressing start.
    2. You might set up a timer instead of timed cook (one is just an alarm, vs one cooks for set amount of time)
    3. Since there is no understanding of how to use power you might accidentally set the power more or less than the one you want.
    4. You can get confused about stop and clear since it is clubbed under one button. If you dont clear after stop, you will not be able to set a new timer the second time.
    5. Accidentally think the digital display is displaying the "oz" and not the timer "seconds".

  • Suggest the improvements to make on the control interface

    1. Reduce the number of buttons | Reduce irrelevant functionality
    2. Change the Labels and Add some description for using buttons
    3. Add different shaped buttons and different colors to add intuition in usage
    4. Add the units inside the digital display and dynamically change it depending on what is being measured
    5. Seperate functionality to different buttons not club them under one button
